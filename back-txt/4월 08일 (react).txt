component를 생성하고 구분하기 위해선 첫글자를 대문자로 사용한다.

component나 props를 불러서 html문서에 컴파일하는 것을 랜더링이라고 한다.

props => 외부로 전달되는 값, 누군가 받아서 처리할 값, 고유명사가 아니라 작명이
 가능하지만 보통 props로 통용하여 사용한다. 
 props는 Objec(개체)로 전달된다.
 props로 전달되는 키의 이름은 내가 정하기 마련 => {props.키의 이름}
 <Header 키의이름="키값" />

{} => 바인딩, 바인딩 안에는 함수만 들어갈 수 있다, 함수호출이 아니라 함수
       function( ) => x / function => o

이벤트,

스테이트, 변하는 자료를 손쉽게 관리하고 상주하는 이유로 일반변수보다 속도가
 월등히 빠르다, 이미 라이브러리로 만들어져있어 가져와서 사용만 하면 된다
 import {useState} from 'react'; 
-> useState라는 라이브러리(함수)를 가져와서 쓴다.
useState는 꼭 App 안에 두어야한다.
기본문법 => let [변수명(개체), 변수변경명(함수)] = useState( );
 변수명의 타입은 object타입, 변수변경명의 타입은 function타입
